# Perftest Task Reference

本文件说明 controller 何时应将请求路由为 `perftest`，以及如何生成稳定的 `task_content`。

## 适用范围

当本轮目标是评估性能表现时，使用 `perftest`。

典型关注点包括：

- 延迟
- 吞吐
- 并发
- p95 / p99
- qps
- 压测稳定性

## 不应使用 `perftest` 的情况

以下情况应回到 `normal`：

- 解释最近一次 perftest 的结果
- 总结性能指标
- 查阅历史压测报告

## `task_content` 生成原则

1. 明确测试对象。
2. 明确性能维度。
3. 明确关键指标。
4. 不写工具与路径。
5. 不写结果。

## 推荐模板

- 针对 `{对象}` 执行性能测试，重点关注 `{指标}`
- 对当前接口执行压测，重点检查 `{指标}`
- 执行性能评估，生成核心指标摘要
